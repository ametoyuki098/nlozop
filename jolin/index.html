自动部署：GitHub Actions

在本节步骤操作之前，请确保你已经按照以上内容成功搭建博客，并且能使用username.github.io访问到自己的博客网站。

Github Actions 简介

Actions是GitHub提供的自动化服务，可以在仓库中定义工作流，当发生特定事件（如 push 代码）时自动执行构建、部署等任务。这就是所谓的CI/CD（持续集成与持续部署）

Actions的核心作用，是让你能直接在GitHub代码仓库中自动化实现完整的 CI/CD（持续集成、持续交付与部署）流程。参考说明

以此教程举例，当使用Github Actions后，我们只需要将配置好的源码推送到目标仓库，GitHub就会自动触发构建并部署。这样一来，就不必在本地反复执行“一键三连命令”
更重要的是，它大大增加了跨设备和无环境的便捷性：你即使没有本地开发环境，只用浏览器也能对文章进行编辑。
